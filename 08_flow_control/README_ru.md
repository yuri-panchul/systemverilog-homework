# Домашнее задание №8

Домашнее задание №8 состоит из 7 упражнений:

- Первое в папке `08_01_halve_tokens_with_flow_control`
- Второе в папке `08_02_double_tokens_flow_control`
- Третье в папке `08_03_generate_tokens_by_number_with_flow_control`
- Четвёртое в папке `08_04_convert_first_to_last_with_flow_control`
- Пятое в папке `08_05_gearbox_1_to_2_fc`
- Шестое в папке `08_06_gearbox_2_to_1_fc`
- Седьмое в папке `08_07_1_a_plus_b_using_double_buffers` или `08_07_2_a_plus_b_using_fifos` или `08_07_a_plus_b_using_fifos_and_double_buffer`

Во всех упражнениях есть секция `Task` с описанием задания на английском языке и местом, где необходимо описать ваше решение.

## Предисловие

В файле Testbench любого из заданий можно убрать комментарий у строк `$dumpfile;` и `$dumpvars;` для генерации `dump.vcd` файла. В файле будут содержаться текстовые описания временной диаграммы, описывающей изменения на всех проводах и регистрах во время симуляции.

Можно воспользоваться командой `gtkwave dump.vcd` для просмотра файла, либо добавить опцию `--wave` или `-w` к скрипту `run_linux_mac` или `run_windows`.
Так же, возможно использовать более современную программу [Surfer](https://surfer-project.org/) для просмотра временных диаграмм.
Surfer доступен на системах Linux, Windows и macOS, а так же в качестве [расширения редактора VS Code](https://marketplace.visualstudio.com/items?itemName=surfer-project.surfer).

## Упражнение 1. Уполовинивание токенов с valid-ready

Задание: Реализуйте модуль, который сократит вдвое количество входящих токенов "1" (логических единиц)
с учётом сигналов `valid` и `ready` для управления потоками приёма (`up_`, то есть `upstream`) и передачи (`down_`, то есть `downstream`).

## Упражнение 2. Удвоение токенов с valid-ready

Задание: Реализуйте модуль, который удваивает каждый входящий токен "1" (логическую единицу)
с учётом сигналов `valid` и `ready` для управления потоками приёма (`up_`, то есть `upstream`) и передачи (`down_`, то есть `downstream`).
Если модуль получает более 100 токенов подряд, то сигнал `up_ready` должен быть установлен в "0" до момента получения `down_ready`.

## Упражнение 3. Генерация заданного количества токенов

Задание: Реализуйте модуль принимающий на вход целое число `N_tokens` и генерирующий `N_tokens` импульсов "1".
Модуль должен использовать сигналы `valid` и `ready` при приёме значения и передаче токенов.

## Упражнение 4. Конвертация сигнала начала в сигнал конца с valid-ready

Задание: Реализуйте модуль для конвертации сигнала, обозначающего начало передачи текущего пакета,
в сигнал, сигнализирующий о конце передачи прошлого пакета с учётом сигналов `valid` и `ready`
для управления потоками приёма и передачи.

Содержимое передаваемых данных должно оставаться неизменным.

## Упражнение 5. Редуктор одинарной размерности в двойную

Задание: Реализуйте модуль, который на основе двух приходящих подряд токенов одинарной размерности
будет формировать один выходящий токен двойной размерности. Модуль должен использовать сигналы `valid` и `ready`
для управления потоками приёма и передачи.

Пример:
"01", "10" => "0110"

## Упражнение 6. Редуктор из двойной размерности в одинарную

Задание: Реализуйте модуль, который на основе одного приходящего токена двойной размерности
будет формировать два выходящий подряд токена одинарной размерности. Модуль должен использовать сигналы `valid` и `ready`
для управления потоками приёма и передачи.

Пример:
"0110" => "01", "10"

## Упражнение 7. Задача "A + B" с использованием буферов

В данном упражнение реализуется схема примерно описанная в статье "[FIFO для самых маленьких](https://habr.com/ru/articles/646685/)" в Примере 3. Две FIFO на входах операции сложения выравнивают результаты по времени и результат складывается в двойной буфер на выходе формулы.

Задание: Соедините все внутренние модули упражнения используя, в том числе, внешние сигналы `valid` и `ready`.
Заготовки всех присваиваний (assign) приведены в комментариях секции `Task`, необходимо реализовать логику формирования сигналов после `=`.
